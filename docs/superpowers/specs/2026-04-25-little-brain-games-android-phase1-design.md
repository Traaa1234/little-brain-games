# Little Brain Games — Android App, Phase 1

**Date:** 2026-04-25
**Phase:** 1 of 2 (Phase 2 = Play Store listing/submission, separate spec)
**Deliverable:** A signed Android App Bundle (`.aab`) of `kids-game/index.html`, ready for Play Store upload.

## Purpose

Wrap the existing single-file kids' learning game as a real Android app: tap-an-icon launch, fullscreen, offline, supports the existing four games (Memory, Pattern, Puzzle, Logic) and the file-upload puzzle feature without modification.

This phase produces the technical artifact. Phase 2 handles the paperwork required to actually publish to the Play Store (privacy policy, store listing assets, content rating, Designed for Families enrollment, Data Safety form, $25 dev signup, review submission).

## Scope

**In scope (Phase 1):**

- Capacitor project that wraps the HTML
- App identity: name, package ID, version
- Adaptive icon (foreground + background layers) at all required densities
- Static splash screen
- Auto-rotation (portrait + landscape both work without losing game state)
- Signed release AAB build pipeline
- Debug build runnable on emulator + real device

**Out of scope (Phase 2 or later):**

- Privacy policy text and hosting
- Play Store listing assets (screenshots, feature graphic, copy)
- Content rating questionnaire
- Designed for Families enrollment / COPPA disclosure
- Data Safety form
- $25 dev account signup
- AAB upload, review, release
- iOS app (different platform)
- In-app updates / OTA web asset updates
- Native features beyond what the WebView already provides (camera, Bluetooth, push, etc.)

## Architecture

```
android-app/                                 ← new top-level directory in this repo
├── android/                                 ← Android Studio project (generated by Capacitor)
│   └── app/
│       ├── build.gradle                     ← versionCode, signing config
│       └── src/main/
│           ├── AndroidManifest.xml          ← orientation, configChanges, theme
│           ├── res/
│           │   ├── mipmap-anydpi-v26/       ← adaptive icon XMLs
│           │   ├── mipmap-{m,h,xh,xxh,xxxh}dpi/  ← rasterized icons
│           │   ├── drawable/                ← splash drawable
│           │   ├── values/colors.xml        ← cream + coral references
│           │   ├── values/strings.xml       ← app_name = "Little Brain Games"
│           │   └── values/styles.xml        ← AppTheme + SplashTheme
│           └── assets/public/               ← HTML lives here at build time (sync'd from ../../../../www)
├── www/                                     ← Capacitor web assets
│   └── index.html                           ← copy of ../../kids-game/index.html
├── capacitor.config.json                    ← name, packageId, plugin config
├── package.json                             ← @capacitor/core, @capacitor/android, etc.
└── README.md                                ← build instructions
```

`www/index.html` is a build-time **copy** of `kids-game/index.html`, not a symlink (Windows symlinks are awkward). The build script copies before each `cap sync`.

## Manifest Tweaks

In `android/app/src/main/AndroidManifest.xml` for the main `<activity>`:

- `android:screenOrientation="fullSensor"` — auto-rotate, both orientations.
- `android:configChanges="orientation|screenSize|keyboardHidden|smallestScreenSize|screenLayout|uiMode"` — the activity handles rotation itself; the WebView is **not** recreated. This preserves game state (a half-completed jigsaw, current memory matches, etc.) across device rotation.
- `android:resizeableActivity="true"` — split-screen / freeform window mode renders correctly.
- `android:hardwareAccelerated="true"` — already default; explicit for clarity.

No new permissions are added. The WebView's file-chooser flow (used by "Upload picture") triggers Android's system picker without requiring an `android.permission.READ_EXTERNAL_STORAGE` declaration on modern Android.

## Icon Design

**Adaptive icon** (Android 8+, API 26+):

- **Background layer (`ic_launcher_background.xml`):** solid coral fill `#F4A261`. Vector drawable, no PNG.
- **Foreground layer (`ic_launcher_foreground.xml`):** a friendly, original "thinking head" silhouette in warm cream `#FFF8EC` — a rounded blob outline (suggesting both a brain and a cute character head) with two small darker dots for eyes and a gentle smile. Three small accent dots (warm yellow `#E9C46A`, sage green `#A8C686`, peach `#F6CBA3`) clustered in the upper area suggest the variety of mini-games. Vector drawable. The art is **original** — no use of system emojis or copyrighted/trademarked imagery.
- **Safe zone:** all foreground content within the inner 66 dp circle of the 108 dp working area, so launchers can crop to circle/squircle/rounded-square without clipping the design.

Adaptive icons are declared via `mipmap-anydpi-v26/ic_launcher.xml` and `ic_launcher_round.xml`, both pointing at the foreground + background pair. Legacy icons (PNG) are also exported at the five density buckets (`mdpi` 48, `hdpi` 72, `xhdpi` 96, `xxhdpi` 144, `xxxhdpi` 192) for older devices, by rasterizing the composed adaptive icon.

**Store-listing icon (Phase 2 input):** the same composed art rendered at 512×512 PNG, on a cream `#FFF8EC` background, padded so the foreground sits inside an inner 80% region. Saved to `android-app/store-assets/icon-512.png`.

## Splash Screen

Static splash (no animation, no spinner):

- `drawable/splash.xml`: a `layer-list` with cream `#FFF8EC` color fill and a centered icon drawable.
- `values/styles.xml` defines `AppTheme.Splash` extending `Theme.AppCompat.NoActionBar` with `android:windowBackground = @drawable/splash` and `android:windowFullscreen = true`.
- The launcher activity uses this theme; on first frame after `setContentView`, the Capacitor SplashScreen plugin fades it out (1500 ms duration per config above).

## Signing

A single release keystore is generated once, **before** the first AAB build, and **must be preserved forever** — losing it means losing the ability to ship updates to the same Play Store listing.

Steps in the implementation plan:

1. Generate keystore using `keytool`:
   ```
   keytool -genkey -v -keystore littlebraingames-release.jks \
     -keyalg RSA -keysize 2048 -validity 10000 -alias littlebraingames
   ```
2. Store keystore at `%USERPROFILE%\.android\keystores\littlebraingames-release.jks` (Windows-friendly path; outside the repo).
3. Add the path + passwords to `android-app/.env.local` (gitignored), with placeholder values committed in `android-app/.env.example`.
4. `android/app/build.gradle` reads env vars and configures the `release` signing config; falls back to `null` if vars missing (debug builds still work).
5. Document the keystore backup procedure in `android-app/README.md` — the user copies the `.jks` file and credentials to a password manager / cloud backup. **This is a critical one-time step.**

## Build Outputs

| Goal | Command | Output |
|------|---------|--------|
| Sync web assets after editing `kids-game/index.html` | `cd android-app && npm run sync` | `android/app/src/main/assets/public/index.html` updated |
| Build debug APK | Run from Android Studio (Build → Build APKs → debug) | `android/app/build/outputs/apk/debug/app-debug.apk` |
| Build release AAB (Play Store) | `cd android-app/android && gradlew.bat bundleRelease` | `android/app/build/outputs/bundle/release/app-release.aab` |
| Run on emulator/device | `cd android-app && npx cap run android` | App installs and launches |

`npm run sync` is a small script defined in `android-app/package.json` that copies `../kids-game/index.html` into `www/` and then runs `npx cap sync android`.

## Testing Plan

Manual verification (no automated test framework added in Phase 1):

1. **Emulator (Pixel 7, API 34):**
   - All 4 games launch, render, score correctly
   - Audio plays after first user gesture
   - Drag-drop in jigsaw works with mouse/trackpad input
   - File-upload picker opens and accepts an image
   - Self-test in console (`window.__runSelfTest()`) reports all green

2. **Real Android tablet over USB (any Android 7+ device):**
   - All 4 games behave the same as emulator
   - Pinch-rotate the device → orientation flips, **game state preserved** (verified by leaving a half-finished jigsaw mid-rotate)
   - Audio plays through device speaker
   - Touch drag of jigsaw pieces is responsive
   - File picker opens device gallery; uploaded picture renders as the puzzle scene

3. **Phone in portrait (any Android 7+ phone):**
   - Game area scales correctly (existing responsive CSS at `@media (max-width: 600px)`)
   - All games playable with thumb input

If any of the above fails, fix and re-test before declaring Phase 1 complete.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Lost release keystore | One-time backup to password manager + cloud, documented in README |
| WebView differences across Android versions causing rendering bugs | Min SDK 24 (Chromium WebView), test on emulator + real device |
| File picker doesn't open in WebView | Capacitor enables `setAllowFileAccess(true)` and `WebChromeClient.onShowFileChooser` by default — verified during testing |
| AudioContext blocked before first gesture | Existing code (`audio.ensureCtx()` called inside game click handlers) already handles this |
| Game state lost on rotation | `configChanges` on the manifest prevents activity recreation — verified during testing |
| Capacitor or @capacitor/android version drift | Pin exact versions in `package.json` |

## Success Criteria

Phase 1 is complete when:

1. `gradlew.bat bundleRelease` produces a signed `.aab` without errors
2. The signed AAB installs and runs on at least one real Android tablet
3. All four games + file upload work on that device
4. Rotation does not lose game state
5. The keystore is backed up outside the repo
6. `android-app/README.md` documents the build, sync, and keystore-backup steps clearly enough that the user can rebuild without re-asking

Phase 2 (Play Store paperwork + submission) starts only after these are met.
