## Purpose

Wrap the existing single-file kids' learning game as an iOS app that runs natively on iPhone (and works on iPad via the iPhone-app-on-iPad fallback). Capacitor wraps the same `kids-game/index.html` we already use for Android, so feature parity is automatic. This phase produces the signed technical artifact. Phase 2 handles App Store metadata, Made for Kids enrollment, paid app agreement, and review submission.

Capacitor uses CocoaPods to manage native iOS dependencies. The first `npx cap add ios` runs `pod install` inside `ios/App/` which downloads CocoaPods + Capacitor's iOS native code (~30 MB). Requires CocoaPods installed on the Mac (`sudo gem install cocoapods` if missing).

## Icon Generation

iOS requires PNG icons at 17 unique size+scale combinations:

| Use | Size (pt) | @1x | @2x | @3x |
|-----|-----------|-----|-----|-----|
| iPhone Notification | 20 | вЂ” | 40px | 60px |
| iPhone Settings | 29 | вЂ” | 58px | 87px |
| iPhone Spotlight | 40 | вЂ” | 80px | 120px |
| iPhone App | 60 | вЂ” | 120px | 180px |
| iPad Notification | 20 | 20px | 40px | вЂ” |
| iPad Settings | 29 | 29px | 58px | вЂ” |
| iPad Spotlight | 40 | 40px | 80px | вЂ” |
| iPad App | 76 | 76px | 152px | вЂ” |
| iPad Pro App | 83.5 | вЂ” | 167px | вЂ” |
| App Store | 1024 | 1024px | вЂ” | вЂ” |

Approach: provide Xcode with a single 1024Г—1024 master, and use Xcode's built-in **AppIcon Asset Catalog** which auto-generates all sizes (since Xcode 14, "Single Size" option). We supply `store-assets/icon-1024.png` (upscaled from the existing `icon-512.png` via Image Magick or the Mac's built-in `sips`) and drag it into the asset catalog.

The existing icon-512 is generated from the same vector source as the Android adaptive icon. To upscale to 1024 cleanly, we re-render the icon SVG (we have one from Android Studio) at 1024Г—1024. If the SVG isn't readily available, doubling the existing 512 PNG with bilinear resampling is acceptable for kids'-app icon quality вЂ” text is minimal and the design is forgiving.

## Splash Screen

iOS uses a **Launch Storyboard** for the splash screen. Capacitor's default storyboard shows a centered image on a colored background. We:

1. Set the storyboard's background color to cream `#FFF8EC` (matches Android)
2. Place the app icon centered (use the 60pt @3x = 180px PNG we'll have)
3. Configure the SplashScreen plugin to overlay for ~1.5s on launch (matches Android)

## Code Signing

Apple uses certificates + provisioning profiles instead of keystores. Modern Xcode (15+) handles this with **Automatic signing**:

1. In Xcode в†’ Project navigator в†’ click `App` (project root) в†’ Signing & Capabilities tab
2. Check **Automatically manage signing**
3. Team: select your Apple Developer team (Individual, your name)
4. Bundle Identifier: `app.kidlearn.tinygenius`
5. Xcode auto-creates the certificate + provisioning profile and stores them in your keychain

For release builds Xcode will prompt to sign with a **Distribution Certificate**. First time, Xcode generates this and uploads it to Apple Developer portal automatically.

Equivalent of Android's keystore-backup discipline: nothing to back up manually вЂ” Apple stores the signing certificate server-side. The user's local keychain holds a private key that can be regenerated if lost (Apple revokes the old one, issues new). Lower stakes than the Android keystore.

   - Rotation flips, **game state preserved** through rotation (verified mid-jigsaw)
   - Audio plays through device speaker
   - Touch drag of jigsaw pieces is responsive
   - Photo picker opens iOS-native picker; uploaded picture renders as the puzzle scene
   - Pinch-zoom works on the puzzle (Capacitor allows this since iOS WebView respects viewport meta tags)
   - Status bar is dark text on cream вЂ” readable

3. **TestFlight (internal):**
   - Upload archive to App Store Connect
   - Add the developer's own Apple ID as an internal tester
   - Install via TestFlight app on iPhone
   - Verify the experience matches USB-installed build (no signing or asset issues)

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `pod install` fails because CocoaPods isn't on the Mac | Run `sudo gem install cocoapods` once before `npx cap add ios` |
| Xcode signing errors (no team selected, missing capability) | Documented step-by-step in Phase 1 implementation plan |
| Apple Developer account locked / not enrolled in Paid Apps Agreement | User does this in parallel вЂ” doesn't block dev/build, only blocks store submission |
| Photo picker text rejected by Apple reviewers as too vague | The proposed text explicitly names the trigger ("only when you tap 'Upload picture'") and the data flow ("stays on your device") вЂ” Apple's pattern for compliant strings |
| `.ipa` upload to App Store Connect fails because Bundle ID isn't registered with Apple | Pre-register the Bundle ID in Apple Developer portal в†’ Identifiers before first archive |
| Status bar invisible on some iPhones with dark mode auto-enabled | Use `style: "DARK"` (dark content) which respects iOS appearance system |
| WebView audio doesn't play on iOS until user gesture | Existing `audio.ensureCtx()` already handles this via click handlers |
